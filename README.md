# Jewelry On Hand

Мобильный MVP для AR-примерки кольца на руке на стеке `Next.js + TypeScript + MediaPipe + Three.js`.

## Что уже реализовано

- стартовый экран с CTA и переходом на `/try-on`
- полноэкранный экран примерки с live-видео и прозрачным `canvas`
- клиентский runtime для камеры, MediaPipe Hand Landmarker и Three.js
- автоматическая привязка к безымянному пальцу одной наиболее уверенно найденной руки
- сглаживание позиции, поворота и масштаба
- короткое удержание последней позы при краткой потере трекинга
- fallback-кольцо, если `GLB`-модель пока не добавлена
- юнит-тесты для `RingPoseSolver`

## Где лежит модель

Основной контракт для модели:

`public/models/demo-ring.glb`

Если файла пока нет, экран всё равно будет работать и покажет процедурное демо-кольцо.

## Установка и запуск

Если используете `nvm`:

```bash
nvm install 22.13.0
nvm use 22.13.0
```

Установка зависимостей:

```bash
npm install
```

Запуск dev-сервера:

```bash
npm run dev
```

После этого откройте:

`http://localhost:3000`

Полезные команды:

```bash
npm run lint
npm test
npm run build
```

## Как добавить свою GLB-модель

Сначала проверьте расширение: нужен именно `.glb`, не `.gbl`.

1. Положите свою модель в папку `public/models/`.
2. Например, если файл называется `my-ring.glb`, путь будет таким:

```text
public/models/my-ring.glb
```

3. Обновите конфиг ассета в `lib/ar/jewelryAssets.ts`:

```ts
export const demoRingAsset: JewelryAsset = {
  anchorStrategy: "ringFinger",
  baseScale: 1.0,
  id: "demo-ring",
  modelUrl: "/models/my-ring.glb",
  modelScale: 1.0,
  name: "Demo Ring",
  positionOffset: [0, 0, 0],
  rotationOffset: [0, 0, 0],
  type: "ring"
};
```

4. Если кольцо слишком большое, маленькое или повернуто не так, подстройте эти поля:

- `baseScale` отвечает за привязку к размеру пальца
- `modelScale` отвечает за прямой размер именно вашей GLB-модели
- `rotationOffset` отвечает за поворот по осям `[x, y, z]` в радианах
- `positionOffset` отвечает за локальное смещение модели относительно вычисленной точки привязки

Стартовые практические правила:

- если модель в целом кажется слишком маленькой или слишком большой независимо от пальца, правьте `modelScale`
- если кольцо сидит на пальце неестественно относительно его толщины, правьте `baseScale`
- если кольцо слишком большое, уменьшайте `modelScale` шагом `0.25-0.5`
- если кольцо слишком маленькое, увеличивайте `modelScale` шагом `0.25-0.5`
- если модель лежит боком, сначала попробуйте `rotationOffset: [Math.PI / 2, 0, 0]`
- если модель уезжает вперёд или назад относительно пальца, правьте `positionOffset`

После замены модели перезапустите dev-сервер или просто обновите страницу, если сервер уже запущен.
